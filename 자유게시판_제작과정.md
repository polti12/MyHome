# 🛠️ 자유 게시판(COMMUNITY) 제작 과정 가이드

데이터베이스(DB) 서버 없이 순수 프론트엔드 기술(HTML/CSS/JS)과 브라우저의 내부 저장소(localStorage)를 활용하여 자유 게시판을 구축하는 단계별 제작 과정입니다.

---

## 1단계: 화면 구조 설계 (HTML)
사용자가 이름, 이메일, 제목, 내용을 입력할 수 있는 **게시글 등록 폼** 영역과, 등록된 게시물 및 답글이 노출되는 **리스트 영역**을 `index.html`에 추가합니다.

```html
<!-- 게시글 등록 폼 영역 -->
<div class="board-form-card">
    <form id="post-form" class="board-form">
        <input type="text" id="name" required placeholder="이름">
        <input type="email" id="email" required placeholder="이메일">
        <input type="text" id="title" required placeholder="제목">
        <textarea id="content" required placeholder="내용"></textarea>
        <button type="submit">게시글 등록</button>
    </form>
</div>

<!-- 게시글 목록 렌더링 영역 -->
<div class="board-list-area">
    <div id="post-list" class="post-list">
        <!-- 자바스크립트를 통해 HTML 요소가 이곳에 생성됩니다. -->
    </div>
</div>
```

---

## 2단계: 디자인 시스템 적용 (CSS)
기존 포트폴리오의 어둡고 사이버틱한 테마에 어울리도록, 입력 폼과 버튼, 그리고 게시글 카드 테두리에 스타일을 부여합니다. 기존 `style.css` (현재 `main.css`)에 다음 내용을 추가했습니다.

```css
/* 반투명한 어두운 배경과 테두리 적용 */
.board-form input, .board-form textarea {
    width: 100%;
    padding: 15px;
    background: rgba(0, 0, 0, 0.5); /* 힌트: 투명도 적용 */
    border: 1px solid rgba(255, 255, 255, 0.1);
    color: white; /* 텍스트 흰색 처리 */
    border-radius: 4px;
}

/* 포커스시 테두리 강조 효과 */
.board-form input:focus {
    border-color: var(--primary-blue);
    box-shadow: 0 0 0 2px rgba(0, 163, 255, 0.2);
}
```
*핵심: 브라우저 캐시 문제를 피하기 위해, 개발 도중에는 CSS 파일명을 `main.css`처럼 변경해주거나 강력 새로고침을 해야 합니다.*

## 3단계: 로컬 스토리지 핵심 데이터 로직 (JavaScript)
데이터베이스가 없는 환경에서는 브라우저의 캐시 메모리인 `localStorage` 영역에 모든 데이터를 문자열(JSON)로 저장합니다. 

### ① 게시글 등록하기
사용자가 '등록' 버튼을 누르면 폼의 데이터를 가져와 묶습니다.
```javascript
const postForm = document.getElementById('post-form');

postForm.addEventListener('submit', (e) => {
    e.preventDefault(); // 새로고침 방지
    
    // 1. 새 객체 생성 (사용자 입력값)
    const newPost = { 
        name: document.getElementById('name').value, 
        email: document.getElementById('email').value, 
        title: document.getElementById('title').value, 
        content: document.getElementById('content').value, 
        date: new Date().toISOString(), 
        replies: [] 
    };
    
    // 2. 기존 스토리지 목록을 불러와서 배열에 추가
    const posts = JSON.parse(localStorage.getItem('free_board_posts')) || [];
    posts.push(newPost);
    
    // 3. 다시 스토리지에 문자열로 변환하여 저장
    localStorage.setItem('free_board_posts', JSON.stringify(posts));
    loadPosts(); // 목록 새로고침
});
```

### ② 화면에 렌더링하기 (불러오기)
저장소에 보관된 게시물 리스트(`posts`)를 꺼내와서 역순(최신순)으로 화면에 그려줍니다.
```javascript
function loadPosts() {
    const posts = JSON.parse(localStorage.getItem('free_board_posts')) || [];
    const postList = document.getElementById('post-list');
    postList.innerHTML = ''; // 초기화

    // 최신 글을 먼저 보이게 하기 위해 배열 순서를 뒤집음
    posts.slice().reverse().forEach((post, index) => {
        const postEl = document.createElement('div');
        
        // 주의: 해킹(XSS) 방지를 위해 escapeHTML() 함수를 함께 사용합니다.
        postEl.innerHTML = `
            <h4>${post.title}</h4>
            <p>${post.content}</p>
        `;
        postList.appendChild(postEl);
    });
}
```

### ③ 보안 방어 적용 (XSS 대비)
게시판의 경우 사용자가悪의적인 스크립트 코드(`<script>얼럿!</script>`)를 내용에 적어서 게시할 수 있습니다. 이를 방지하기 위해 꺽쇠나 특수문자를 단순한 문자로 변환하는 방어 코드를 필수적으로 넣습니다.
```javascript
function escapeHTML(str) {
    return (str || '').toString()
        .replace(/&/g, '&amp;')
        .replace(/</g, '&lt;')
        .replace(/>/g, '&gt;');
}
```

## 요약
이 3가지 단계(화면 레이아웃 구성 -> CSS 디자인 매칭 -> localStorage 데이터 핸들링)를 거쳐, 서버 비용 전혀 없이도 사용자와 상호작용 할 수 있는 게시판(Community) 섹션 제작이 완성되었습니다.
