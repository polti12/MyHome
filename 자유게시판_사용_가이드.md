# 📝 포트폴리오 내 자유 게시판 구축 및 사용 가이드

본 문서에서는 새롭게 추가된 '자유 게시판(COMMUNITY)' 섹션에 대한 기술 스택과 테스트 방법을 안내해 드립니다.

## ✨ 들어가며
요청하신 대로, 백엔드 서버나 별도의 데이터베이스 없이 **오직 웹 브라우저의 '로컬 스토리지(localStorage)' 기능**만을 활용하여 자유 게시판을 포트폴리오 안에 완전히 통합하였습니다. 어두운(Dark) 테마의 기존 포트폴리오 디자인 시스템과 어울리도록 세련되고 직관적인 UI로 디자인 되었습니다.

## 🛠 주요 기능 및 로직
1. **게시글 등록 폼 (.board-form)**
   - 이름, 이메일, 제목, 내용을 입력하여 새로운 게시글을 작성할 수 있습니다.
   - 폼 데이터는 `script.js` 내에서 JSON 객체로 변환되어 브라우저 내부에 저장됩니다.

2. **게시글 목록 렌더링**
   - 최신 게시글이 가장 상단에 배치되도록 역순(Reverse)으로 화면에 표시됩니다.
   - 저장된 데이터가 없는 경우 우아하게 "등록된 게시글이 없습니다"라는 초기 메시지를 보여줍니다.

3. **답글(댓글) 기능 적용**
   - 각 게시글마다 종속적인 '답글 리스트'를 가지며, 새로운 폼을 통해 답글을 즉시 추가할 수 있습니다.

4. **단일 스토리지 데이터 유지성**
   - `window.localStorage` 영역에 저장되므로, 실수로 브라우저를 닫거나 새로고침(F5)을 하더라도 남긴 질문과 답글이 그대로 영구 보존됩니다.
   
5. **보안 (XSS 방어)**
   - 사용자가 스크립트(태그)를 주입할 수 없도록 `escapeHTML` 처리 필터 함수를 도입하여 안전성을 높였습니다.

---

## 🚀 어떻게 구동 및 테스트하나요?

1. 바탕화면의 `AG WORK` 폴더로 이동합니다.
2. 폴더 내에 있는 **`index.html` 파일을 더블 클릭**하거나 브라우저 창에 드래그하여 엽니다.
3. 상단 내비게이션 바에서 **[COMMUNITY]** 메뉴를 클릭하여 게시판으로 이동합니다!
4. 브라우저에서 직접 게시글 작성 버튼과 답글 등록 버튼을 누르며 데이터 보존 여부를 테스트해 보세요.

> 💡 **참고**: CSS 오류 방지를 위해 캐시가 걸리지 않은 새로운 파일인 `main.css`를 생성하고 연동해 두었습니다. 디자인이 정상적으로 나타납니다!
