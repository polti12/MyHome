# 포트폴리오 리팩토링 및 환경 구축 히스토리 (개발 일지)

## 📌 1. 데이터베이스 기술 스택 검토
* **요청 사항:** 구글에서 제공하는 무료 데이터베이스 옵션 및 AI/서비스 개발 환경에 적합한 데이터베이스 스택 조사.
* **작업 결과:** Firebase Realtime Database와 Cloud Firestore의 구조적 장단점을 비교 분석하고, 현재 개발 중인 포트폴리오 게시판에 가장 적합하고 확장성이 뛰어난 **Cloud Firestore**를 메인 데이터베이스로 선정하여 초기 환경 설정을 제안함.

## 📌 2. Firebase Cloud Firestore 연동 및 개발 환경 구축
* **요청 사항:** Firebase를 포트폴리오 게시판에 연동하는 단계별 가이드라인 설계 및 로컬/NPM 빌드 환경에서의 패키지 적용 방법 문의.
* **작업 결과:** Vanilla JS 및 HTML 환경에 맞는 CDN 모듈 임포트 방식을 채택하여 라이브러리를 구축함. 프로젝트 대시보드에서 Web App API Key와 `firebaseConfig`를 발급받아 프로젝트에 심는 일련의 과정을 상세히 가이드함.

## 📌 3. LocalStorage 기반 게시판의 완전한 클라우드 마이그레이션
* **요청 사항:** 설정이 완료된 Firebase 환경을 기존 `board.html` 및 `board.js` 로직에 전면 적용 및 리팩토링.
* **작업 결과:** 기존 브라우저 임시 캐시(`localStorage`)에만 의존하던 글 작성, 삭제, 수정, 답글, 추천 기능 로직을 완전히 덜어내고, Firestore SDK(`addDoc`, `getDocs`, `updateDoc` 등)를 100% 활용하는 비동기(Async) 클라우드 로직으로 재작성하여 코드를 성공적으로 덮어쓰기 완료함.

## 📌 4. Vercel 배포 후 연동 장애(CORS 및 권한) 트러블슈팅
* **요청 사항:** 코드를 Vercel에 업로드한 후 게시판 글쓰기 버튼이 응답하지 않거나, 데이터가 저장되지 않는 문제에 대한 원인 분석 및 해결.
* **작업 결과:** 
  1. `<script type="module">` 선언 시 발생하는 로컬 `file://` 프로토콜의 CORS 제약을 원인으로 파악하여 정확한 서버 환경(Live Server) 테스트 방법 안내.
  2. 사용자가 실수로 Firestore 대신 구형 'Realtime Database'를 개설하여 발생했던 `.read` / `.write` 규칙 권한 에러의 핵심을 찾아냄. 이후 Firestore Database 기반으로 정확히 방향을 수정하여 접속 장애를 완벽하게 해결함.

## 📌 5. 디자인 시스템 및 레이아웃 대규모 개편 (UI/UX)
* **요청 사항:** 대조가 심한 검정 배경색의 시각적 완화, 일관성 없는 폰트로 인한 가독성 저하 해결, 및 동영상 재생 에러 수정.
* **작업 결과:**
  1. **색감(Color):** 눈이 피로한 완전 검정색 대신 부드럽고 전문적인 분위기를 띄는 네이비 슬레이트(Slate-900) 톤으로 스타일의 전반적인 분위기 전면 개편.
  2. **폰트(Typography):** 폰트 다운로드 실패 충돌을 방지하기 위해 가장 안정적인 `@font-face` 방식으로 가독성이 뛰어난 한국어 폰트인 **'Pretendard'**를 전역 적용함.
  3. **비디오 호환성:** `.MOV` 확장자가 브라우저에서 유발하는 자동 재생 렌더링 에러를 우회할 수 있도록 Video 렌더링 태그 코드를 교정함.

## 📌 6. 뷰포트(Viewport) 및 반응형 그리드 핏(Fit) 최적화
* **요청 사항:** 노트북/모니터 화면에서 카드가 과도하게 확대되어 표기되거나, 검정색 여백이 튀어 한 화면에 정보가 온전히 들어오지 않는 디자인 결함 해결.
* **작업 결과:**
  1. 세로형 영상과 테크니컬 로그 텍스트의 그리드 비율(`1fr : 2.5fr`)을 재조정하여 무의미하게 버려지는 검은색 화면 여백을 소거함.
  2. 핵심 역량 카드(Technical Mastery) 파트가 한 화면에 **2x2 배열**로 깔끔하게 꽉 차서 들어올 수 있도록 `max-height`를 제한하고 패딩을 시원하게 최적화.
  3. 면접관의 시선이 집중되는 '테크니컬 로그(Technical Log)' 텍스트 내의 중요 성과 키워드(응력 분산, 150% 극대화, 오차 0.1mm 등)들을 황금색 볼드체로 하이라이트 처리. 추가로 CSS 덮어쓰기 중 발생한 폰트어썸(FontAwesome) 아이콘 깨짐 버그까지 완벽하게 보수 완료함.
